Evolutionary LLM-Assisted Algorithm Optimization Inspired by AlphaEvolve

Proposal Page
Recent advances in automated algorithm design have demonstrated that evolutionary systems combined with large language models (LLMs) can iteratively improve program quality through evaluation-driven refinement. AlphaEvolve, developed by Google, illustrates how an orchestrated pipeline can generate candidate programs, evaluate them using automated metrics, and progressively improve solutions across generations. The key insight behind such systems is that algorithm optimization can be framed as a search problem guided by fitness evaluation. By continuously selecting and refining higher-performing candidates, the system converges toward improved solutions without requiring direct human redesign of the algorithm at every step.
Inspired by this framework, our project aims to design and implement a simplified evolutionary agent system, called AlgoChat, that integrates LLM-based candidate generation with evolutionary optimization. The system accepts user-provided input in the form of source code, pseudocode, or a structured algorithm description. It then generates candidate program variants by modifying or refining portions of the original solution. Candidate generation strategies include random perturbations, template-based code replacement, rule-based mutation (e.g., swapping or editing code segments), and mandatory LLM-guided refinement via prompt-based improvement. By combining stochastic mutation with LLM-guided reasoning, the system explores both random and intelligent search strategies within the solution space.
Each candidate program is evaluated using automated metrics defined by the target problem domain. The primary use case of this project is the Pacman environment, where performance is measured through gameplay simulation. Evaluation metrics include game score, survival time, and number of steps (treated as computational cost). The overall fitness score is defined as:
Fitness=w1⋅score+w2⋅survival_time−w3⋅steps,subject to w1+w2+w3=1.


This formulation enables flexible trade-offs between aggressive scoring behavior, survivability, and efficiency. After evaluation, the evolutionary algorithm applies selection (e.g., single-best or top-k selection) to retain higher-performing candidates. The process repeats for nnn generations, allowing users to observe measurable improvement in fitness scores over time.
To analyze system effectiveness, we will compare three experimental settings: (1) no evolution (single-shot LLM generation), (2) random mutation without LLM guidance, and (3) LLM-guided mutation within the evolutionary loop. Fitness progression across generations will be visualized through a user interface that displays the number of generations, best fitness score, final best solution, and explanations of candidate generation and selection operations. Additionally, execution time considerations will be addressed through caching mechanisms, including reuse of high-quality candidates and vector database storage of previously evaluated solutions to accelerate search. Through these components, the proposed system demonstrates how evolutionary computation and LLM-based refinement can be integrated into a unified algorithm optimization assistant.


System Framework
The proposed system, AlgoChat, is a unified evolutionary agent that integrates large language model (LLM)–guided code generation, automated fitness evaluation, and iterative optimization within a single architecture. The system accepts user-provided code, pseudocode, or structured algorithm descriptions as input through a web-based interface. From this baseline solution, Evolve initializes a candidate population and executes an evolutionary loop consisting of candidate generation, evaluation, selection, and iteration. The system is designed to demonstrate measurable improvement in solution quality across generations while maintaining transparency through fitness visualization and solution tracking.
Candidate generation is performed using multiple mutation strategies to balance exploration and refinement. These strategies include random perturbation of parameters, rule-based modification of code fragments (e.g., swapping statements or altering conditions), template-based replacement from a predefined repository, and mandatory LLM-guided refinement through prompt-based improvement. To support intelligent mutation, Evolve incorporates a local retrieval module built using SentenceTransformers embeddings and a FAISS vector index. Code templates, prior high-performing candidates, and mutation patterns are embedded and stored in the vector database. When generating new candidates, the system retrieves contextually relevant fragments and supplies them to the LLM, enabling guided improvements grounded in previously successful solutions.
Each generated candidate is evaluated using domain-specific automated metrics. In the Pacman use case, evaluation returns three key measurements: score, survival time, and number of steps (treated as cost). The overall fitness value is computed using the weighted formulation:
Fitness=w1⋅score+w2⋅survival_time−w3⋅steps,subject to w1+w2+w3=1.

Candidates are ranked according to fitness, and selection is performed using either single-best retention or top-kkk selection. The evolutionary process repeats for nnn generations, with best fitness values logged and visualized to illustrate convergence behavior and performance improvement over time.
To enhance efficiency, Evolve integrates caching mechanisms within the same vector database infrastructure. Previously evaluated candidates are embedded and stored to prevent redundant computation. If a newly generated candidate is sufficiently similar to an existing stored solution, the system can reuse the cached fitness result instead of re-evaluating the program. This embedding-based reuse mechanism accelerates search while maintaining evolutionary diversity. The computational complexity per generation is approximately O(P×E) where P represents population size and E denotes the cost of executing one evaluation episode. By combining LLM-guided mutation, embedding-based retrieval, automated evaluation, and evolutionary selection within a single cohesive framework, Evolve demonstrates how algorithm refinement can be achieved through structured, iterative optimization.
